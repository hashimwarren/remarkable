# Design Visible Evidence

For every major claim ask:

> What would let the reader see this for themselves?

Use the strongest fitting rung:

1. **Show it:** video, screenshot, demonstration, or before-and-after.
2. **Measure it:** chart, benchmark, analytics, or a data table.
3. **Let someone else say it:** customer quotation, review, or expert testimony.
4. **Explain it:** mechanism, concrete example, or logical demonstration.

Recommend only assets with an evidentiary job. Do not add decorative media as proof. In this beta, create specifications rather than media unless the user explicitly asks for production: screenshot shot lists, recording scripts, chart specifications, data tables, diagram plans, captions, annotations, and alt text.

Place an asset placeholder next to the claim it supports, state what the asset must demonstrate, and remove unused placeholders before final handoff.
